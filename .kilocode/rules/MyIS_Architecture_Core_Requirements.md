# Архитектурные требования к каркасу MyIS

## 1. Требования к идентификаторам
- Все сущности используют GUID или специализированные ID-VO.

## 2. Value Objects
- Обязательные VO: Quantity, Money, DateRange, Status, Version.

## 3. Workflow
- Статусы типизированы.
- Переходы управляются доменной логикой.

## 4. Модульные границы
- Каждый модуль имеет свои сущности.
- Взаимодействие строго через интерфейсы Application.

## 5. AccessChecker
- Должен поддерживать расширяемую Permission-модель.

## 6. Аудит
- Создание/изменение/удаление должно фиксироваться.

## 7. Интеграции
- Все интеграции через абстракции, обязательная обработка ошибок.

## 8. Структура БД
- Таблицы организованы по схемам: core, requests, warehouse и др.
